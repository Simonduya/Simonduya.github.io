程序错误与异常

访问对象的未知属性

在 JavaScript 中，如果一个对象的某个属性为 null 或 undefined，那么在访问该属性的方法或属性时会导致程序抛出异常并停止执行。

使用try...catch可以捕获异常，程序不会中断

使用可选链操作符访问对象属性可以确保即使 对象的属性 为 null 或 undefined，代码也可以正常执行，避免程序抛出异常。

因此，在编写 JavaScript 程序时应该特别注意这种情况，并使用适当的方法来避免出现异常。例如，可以使用可选链操作符 ?. 来避免访问不存在的属性或方法，或者使用条件判断语句来检查数组元素是否存在等。

JavaScript错误类型示例代码

以下是 JavaScript 中错误类型的示例代码，它们都会导致程序抛出异常并停止执行：

  1. EvalError 错误：当使用全局函数 eval() 中的代码存在错误时，JavaScript 抛出 EvalError 错误。
  1. RangeError 错误：当使用超出有效范围的值创建对象或数组时，JavaScript 抛出 RangeError 错误。
  1. ReferenceError 错误：当尝试访问一个不存在的变量或函数时，JavaScript 抛出 ReferenceError 错误。
  1. SyntaxError 错误：当 JavaScript 代码包含语法错误时，例如括号不匹配、缺少分号等，JavaScript 抛出 SyntaxError 错误。
  1. TypeError 错误：当尝试访问一个不存在或不支持的方法或属性时，JavaScript 抛出 TypeError 错误。
  1. URIError 错误：当传递给 URI 相关的全局函数的参数无效时，例如解码无效的 URI 字符串，JavaScript 抛出 URIError 错误。
  1. AggregateError 错误：当 Promise.all() 或 Promise.allSettled() 方法中的一个或多个 Promise 拒绝了（rejected）时，JavaScript 抛出 AggregateError 错误。它包含一个具有所有拒绝原因的数组。
  1. InternalError 错误：当 JavaScript 引擎发生内部错误时，例如无限递归、堆栈溢出等，JavaScript 抛出 InternalError 错误。

总之，这些错误类型都可能导致程序崩溃或停止执行，并且通常需要使用调试工具来找到和解决。在编写 JavaScript 程序时，应该仔细考虑并避免出现这些错误，以确保程序的正确性和稳定性。
