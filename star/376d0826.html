回溯算法

发表于2023-06-10|更新于2023-08-01|let's code
|阅读量:
  • 回溯法就用递归来解决嵌套层数的问题。
  • 回溯法解决的问题都可以抽象为树形结构（N 叉树)

第 77 题. 组合

给定两个整数 n 和 k，返回 1 … n 中所有可能的 k 个数的组合。

示例: 输入: n = 4, k = 2 输出: [ [2,4], [3,4], [2,3], [1,2], [1,3], [1,4], ]

回溯法

思路:

  1. 将所有的组合抽象为一颗树;
  2. for 循环横向遍历每一层, 递归纵向遍历整个树
  3. 需要一个 start 来标识每个处理节点的起点;
  4. 使用一个 res 数组保存所有结果, 使用 path 数组保存每个满足条件的节点;

时间复杂度: O(n * 2^n)

空间复杂度: O(n)

剪枝优化

n = 4, k = 3, 那么从 3 开始搜索就不会满足 k = 3 的条件了;
所以可以在搜索的起始位置进行优化;
例如: n = 4, k = 3, path.length = 0;
目前已经保存的元素: path.length
需要的元素: k - path.length
i = n - (k - path.length) + 1;
i = 4 - (3 - 0) + 1 = 2;
也就是 2 之前进行搜索都是可以的, 但从 3 开始搜索就不满足 k 个元素

216.组合总和 III

找出所有相加之和为 n 的 k 个数的组合。组合中只允许含有 1 - 9 的正整数，并且每种组合中不存在重复的数字。

说明：

所有数字都是正整数。
解集不能包含重复的组合。
示例 1: 输入: k = 3, n = 7 输出: [[1,2,4]]

示例 2: 输入: k = 3, n = 9 输出: [[1,2,6], [1,3,5], [2,3,4]]

思路

相对于组合总数, 这题需要统计总和, 所以加入一个sum变量, 同时递归结束条件改为sum === n && path.length === k;

剪枝

  1. 如果sum已经大于n了, 剩下的元素不需要考虑, 直接return结束当前函数
  2. 元素个数不足的剪枝i <= 9 - (k - path.length) + 1;

17.电话号码的字母组合

力扣题目链接(opens new window)

给定一个仅包含数字 2-9 的字符串，返回所有它能表示的字母组合。

给出数字到字母的映射如下（与电话按键相同）。注意 1 不对应任何字母。

示例:

输入：”23”
输出：[“ad”, “ae”, “af”, “bd”, “be”, “bf”, “cd”, “ce”, “cf”].
说明：尽管上面的答案是按字典序排列的，但是你可以任意选择答案输出的顺序。

思路

  1. 需要一个map来映射数字和字符串
  2. 需要一个startIndex来遍历数字数组
