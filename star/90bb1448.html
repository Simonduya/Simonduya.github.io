深入理解BFC

发表于2023-09-20|更新于2023-09-20|CSS
|阅读量:

什么是 FC

“FC” 是 Formatting Context 的缩写，它指的是一种用于排版和布局 HTML 元素的机制。在 HTML 文档中，元素通常属于一个特定的 Formatting Context。

块级元素的布局通常归属于 Block Formatting Context（BFC）。
这意味着块级元素的盒子（box）会在 BFC 中进行排列和布局。

行内级元素的布局通常归属于 Inline Formatting Context（IFC）。
这意味着行内级元素的盒子会在 IFC 中进行排列和布局。

哪些情况会创建 BFC

MDN 上有整理出在哪些具体的情况下会创建 BFC：
根元素（）
浮动元素（元素的 float 不是 none）
绝对定位元素（元素的 position 为 absolute 或 fixed）
行内块元素（元素的 display 为 inline-block）
表格单元格（元素的 display 为 table-cell，HTML 表格单元格默认为该值），表格标题（元素的 display 为 table-caption，HTML 表格标题默认为该值）
匿名表格单元格元素（元素的 display 为 table、table-row、 table-row-group、table-header-group、table-footer-group（分别是 HTML table、
row、tbody、thead、tfoot 的默认属性）或 inline-table）
overflow 计算值(Computed)不为 visible 的块元素
弹性元素（display 为 flex 或 inline-flex 元素的直接子元素）
网格元素（display 为 grid 或 inline-grid 元素的直接子元素）
display 值为 flow-root 的元素

BFC 有什么用?

在 BFC 中，元素的盒子会在垂直方向上依次排列，即一个挨着一个，不会出现浮动（float）或者重叠（overlap）的情况。这垂直方向上的间距由 margin 属性决定，也就是说，元素之间的距离是通过它们的上下 margin 来控制的。

在同一个 BFC 中，相邻两个盒子的垂直 margin 会发生折叠（margin collapse）的现象。这意味着如果两个相邻的元素都有 margin，它们的垂直 margin 值不会简单地相加，而是取其中较大的那个值作为最终的间距。

在 BFC 中，每个元素的左边缘通常紧挨着包含块的左边缘，这是默认的情况。但也可以通过一些 CSS 属性（例如 float、position 等）来改变元素的位置和布局。

BFC 的作用一：解决折叠问题

我们让两个 box 是不同的 BFC, 那么就可以解决折叠问题。

BFC 的作用二：解决浮动高度塌陷

BFC 解决高度塌陷需要满足两个条件：

  1. 浮动元素的父元素触发 BFC，形成独立的块级格式化上下文（Block Formatting Context）；
  2. 浮动元素的父元素的高度是 auto 的；

BFC 的高度是 auto 的情况下，是如下方法计算高度的

  1. 如果只有 inline-level，是行高的顶部和底部的距离；
  2. 如果有 block-level，是由最底层的块上边缘和最底层
    块盒子的下边缘之间的距离
  3. 如果有绝对定位元素，将被忽略；
  4. 如果有浮动元素，那么会增加高度以包括这些浮动元
    素的下边缘
