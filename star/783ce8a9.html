视口的理解

参考: https://www.quirksmode.org/mobile/viewports2.html

对于 PC 端来说，布局视口（浏览器布局的视口）通常等于视觉视口（我们看到的视口）。这是因为 PC 屏幕通常足够大，能够容纳整个网页的宽度，因此无需进行缩放或调整。

然而，在移动端，情况通常不同。默认情况下，移动端的布局视口（Layout Viewport）通常会大于视觉视口（Visual Viewport）。这是因为移动设备的屏幕相对较小，无法容纳整个网页的宽度，因此浏览器会对页面进行缩放，以便内容适应屏幕大小，从而创建视觉视口。这个视觉视口是用户实际可见的区域，因此用户可以滚动页面来查看其余部分。

布局视口（Layout Viewport）

在移动端，通常情况下，一个最初设计用于 PC 端的网页会以 980px 的宽度来布局页面的盒子和内容。这个以 980px 宽度布局的视口称为布局视口（Layout Viewport）。布局视口是页面的初始布局容器，其默认宽度为 980px。

视觉视口（Visual Viewport）

然而，由于移动设备的屏幕尺寸有限，显示整个 980px 宽度的内容可能导致部分内容在屏幕上无法完整显示。为了让用户能够看到完整的页面内容，移动设备浏览器会默认进行缩放，以便内容适应屏幕大小。这个被用户看到的实际可见区域称为视觉视口（Visual Viewport）。在某些移动浏览器上，用户可以通过按住 Shift 键并滚动鼠标滚轮来手动调整视觉视口的缩放级别。

理想视口（Ideal Viewport）

为了在移动设备上获得更好的用户体验和页面布局，可以通过设置<meta>标签中的 viewport 属性来调整布局视口。这个被调整后的视口，通常被称为理想视口（Ideal Viewport）。理想视口的设置可以包括指定视口的宽度、缩放级别等，以确保页面内容在移动设备上呈现得更为合适。

总结来说，布局视口是初始的页面布局容器，视觉视口是用户实际可见的区域，而理想视口是通过设置 viewport 属性来调整布局视口以适应移动设备的视口，以提供更好的用户体验。

属性 可能的附加值 描述
width 一个正整数，或者字符串 定义 viewport 的宽度。
height 一个正整数，或者字符串 定义 viewport 的高度。未被任何浏览器使用。
initial-scale 一个 0.0 和 10.0 之间的正数 定义设备宽度与 viewport 大小之间的缩放比例。
maximum-scale 一个 0.0 和 10.0 之间的正数 定义缩放的最大值，必须大于等于 minimum-scale，否则表现将不可预测。
minimum-scale 一个 0.0 和 10.0 之间的正数 定义缩放的最小值，必须小于等于 maximum-scale，否则表现将不可预测。
user-scalable yes 或者 no 默认为 yes，如果设置为 no，将无法缩放当前页面。浏览器可以忽略此规则。

例如:
当在 PC 端设置一个 100 * 100 px 的盒子时，如果在移动设备上默认使用较大的布局视口宽度（例如 980px），那么该盒子的大小在视觉视口中会显得明显较小。这是因为布局视口大于移动设备的实际宽度，导致页面内容在视觉视口中被缩小显示。

为了解决这个问题，可以使用 HTML 中的标签来设置 viewport 属性。通过将 width 属性设置为 device-width，相当于告诉浏览器将布局视口的宽度设置为与设备宽度相同，这样视觉视口中的盒子将与布局视口中的大小相一致，不再出现缩小的情况。这有助于确保在移动设备上获得更好的用户体验和一致的布局呈现。

移动端适配方案

在移动端，由于屏幕尺寸多种多样，我们经常需要根据不同的屏幕尺寸来调整元素的大小，包括设置盒子其他尺寸也是类似，比如 padding、margin、border、left，甚至是 font-size 等。为了应对这种多样性，我们可以考虑以下不同的尺寸适配方案：

例如: 我们设置一个 100x100 的盒子
在 375px 的屏幕上显示是 100 x 100;
在 320px 的屏幕上显示是 90+ x 90+;
在 414px 的屏幕上显示是 100+ x 100+;
希望同屏幕的放大(缩小)来放大(缩小)页面的元素

方案一：百分比设置
百分比单位可以用来相对于父元素来设置元素的大小，但在移动端适配中，由于不同属性可能相对于不同的参照物，因此使用百分比来统一尺寸往往会比较复杂，因此不太常见。不同属性的百分比单位相对于不同的元素

方案二：rem 单位 + 动态的 html font-size
使用 rem 单位可以相对于根元素的字体大小来设置元素的尺寸，同时动态调整根元素的字体大小可以实现适应不同屏幕尺寸的效果。这是一种相对常见且灵活的适配方案。

方案三：vw 单位
vw 单位是相对于视窗宽度的百分比单位，它可以用来根据屏幕宽度调整元素大小。这个单位在移动端适配中也经常被使用，特别是对于宽度相关的属性。

方案四：flex 的弹性布局
使用 flex 布局可以使元素根据可用空间自动分配大小，适用于构建灵活的布局。这种方式在移动端适配中也很实用，特别是对于响应式布局的情况。

综合来说，选择哪种尺寸适配方案取决于具体的需求和设计情况。通常，结合 rem 单位、vw 单位和 flex 布局可以创建出适应不同屏幕尺寸的移动端界面，而百分比设置相对较少使用。在实际开发中，根据具体情况灵活选择合适的方案来进行移动端适配是关键。

适配方案 rem + 动态设置 html 的 font-size

rem 单位是相对于 html 元素的 font-size 来设置的，那么如果我们需要在不同的屏幕下有不同的尺寸，可以动态的修改 html 的
font-size 尺寸。
比如如下案例： 1.设置一个盒子的宽度是 1rem； 2.设置不同的屏幕上 html 的 font-size 不同；

屏幕尺寸 HTML 的 font-size 盒子的设置宽度 盒子的最终宽度
375px 37.5px 1rem 37.5px
320px 32px 1rem 32px
414px 41.4px 1rem 41.4px

这样在开发中，我们只需要考虑两个问题：
问题一：针对不同的屏幕，设置 html 不同的 font-size；
问题二：将原来要设置的尺寸，转化成 rem 单位；

动态设置 html 的 font-size

在移动端开发中，有多种方案可以实现根据不同屏幕尺寸动态调整 HTML 的 font-size 尺寸，以适应不同屏幕的需求。

方案一：媒体查询
可以使用媒体查询来针对不同尺寸范围内的屏幕设置不同的 HTML font-size 尺寸。然而，这种方法有以下缺点：

  1. 需要编写大量媒体查询规则，对于多个屏幕尺寸会显得繁琐。
  2. 如果屏幕尺寸发生动态改变，媒体查询不会实时更新 HTML 的 font-size。

方案二：编写 JavaScript 代码
如果需要实时响应屏幕尺寸的变化并相应地调整 font-size，可以使用 JavaScript 来实现。方法如下：

  1. 根据 HTML 的宽度计算出 font-size 的大小，并设置到 HTML 元素上。
  2. 监听屏幕尺寸的实时改变事件，并在变化时重新计算和设置 font-size 大小到 HTML 元素上。

这种方法允许根据屏幕的实时情况进行适配。

方案三：lib-flexible 库
lib-flexible 库实际上是一种 JavaScript 库，它提供了自动计算和设置 HTML font-size 的功能，以便在不同屏幕尺寸下进行适配。

vw 单位替换动态 font-size

使用 vw（视窗宽度的百分比单位）来代替动态设置 HTML 的 font-size 是一个有效的方法，特别是在响应式设计中。通过将 HTML 的 font-size 设置为 10vw，可以使其与 375px 的设计稿基本等比例匹配。

具体来说，将 HTML 的 font-size 设置为 10vw 的效果如下：

当视窗宽度为 375px 时，font-size = 10vw = 37.5px。
当视窗宽度为 320px 时，font-size = 10vw = 32px。
当视窗宽度为 414px 时，font-size = 10vw = 41.4px。
这种方式可以更简单地实现响应式设计，无需手动计算或编写 JavaScript 代码来根据屏幕尺寸调整 font-size。通过将 font-size 设置为相对于视窗宽度的百分比单位，可以在不同屏幕尺寸下自动适应设计稿的比例。

vw 结合 rem 单位完成移动适配

结合 vw 单位和 rem 单位可以非常有效地实现移动端的适配。下面简要说明如何结合使用这两种单位：

  1. 设置 HTML 的 font-size 为 vw 单位： 在 CSS 中，您可以将 HTML 的 font-size 设置为相对于视窗宽度的 vw 单位，例如设置为 10vw，以便在不同屏幕宽度下自动调整字体大小。

  2. 使用 rem 单位来设置其他元素的尺寸： 除了字体大小以外的元素尺寸，如宽度、高度、内边距、外边距等，可以使用 rem 单位来设置。假设您将 1rem 等于 10px，那么在这个基础上，您可以根据设计稿的尺寸来设置元素的大小。

这样，通过将 HTML 的 font-size 设置为 vw 单位，以及使用 rem 单位来设置其他元素的尺寸，您可以实现移动端的适配。在不同屏幕宽度下，字体大小会根据视窗宽度进行调整，而其他元素的尺寸会相对于 HTML 的 font-size 进行适应，从而确保页面在不同屏幕上具有一致的布局和样式。这种组合通常用于响应式设计，使网站能够适应各种移动设备和屏幕尺寸。

vw 和 rem 对比

rem事实上是作为一种过渡的方案，它利用的也是vw的思想。
前面不管是我们自己编写的js，还是flexible的源码；
都是将1rem等同于设计稿的1/10，在利用1rem计算相对于整个屏幕的尺寸大小；
那么我们来思考，1vw不是刚好等于屏幕的1/100吗？
而且相对于rem还更加有优势；
vw相比于rem的优势：
优势一：不需要去计算html的font-size大小，也不需要给html设置这样一个font-size；
优势二：不会因为设置html的font-size大小，而必须给body再设置一个font-size，防止继承；
优势三：因为不依赖font-size的尺寸，所以不用担心某些原因html的font-size尺寸被篡改，页面尺寸混乱；
优势四：vw相比于rem更加语义化，1vw刚才是1/100的viewport的大小;
优势五：可以具备rem之前所有的优点；
vw我们只面临一个问题，将尺寸换算成vw的单位即可；
所以，目前相比于rem，更加推荐大家使用vw（但是理解rem依然很重要）
