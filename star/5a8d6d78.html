promise 简介

在 JavaScript 中，Promise 是一种用于处理异步操作的对象，它可以更优雅地组织和处理异步代码。Promise 有三种状态：pending（进行中）、fulfilled（已成功）和 rejected（已失败）。一旦 Promise 状态变为 fulfilled 或 rejected，它就会被认为是“settled”（已解决），之后它的状态不会再改变。

以下是使用 JavaScript 实现 Promise 的基本步骤：

  1. 创建 Promise 对象：使用 Promise 构造函数来创建一个新的 Promise 对象，它接受一个 executor 函数作为参数。executor 函数在创建 Promise 时会立即执行，它有两个参数：resolve 和 reject，分别用于将 Promise 从 pending 状态转换为 fulfilled 状态和 rejected 状态。

  2. 异步操作：在 executor 函数中，编写处理异步操作的代码。可以是网络请求、文件读取等耗时操作。

  3. 根据异步操作结果：在异步操作完成后，根据操作的成功或失败情况，调用 resolve 和 reject 来改变 Promise 的状态。

  4. 处理结果：使用 Promise 的 then 方法来处理异步操作的结果，通过 then 方法的回调函数获取异步操作的成功结果，并使用 catch 方法来捕获异步操作的失败情况。

问题关键: 何时执行 resolve, 何时执行 reject

介绍

当涉及并行处理多个异步任务时，JavaScript 中的 Promise.all()方法是一个非常有用的工具。它接受一个由 Promise 对象组成的数组，并返回一个新的 Promise 对象。这个新的 Promise 对象将在数组中的所有 Promise 都已经成功解决（resolved）时才会被解决，或者在数组中的任何一个 Promise 被拒绝（rejected）时被拒绝。

如果在数组中的任何一个 Promise 被拒绝，那么 Promise.all()返回的 Promise 对象也会被拒绝，并且它的拒绝原因将是第一个被拒绝 Promise 的拒绝原因。

需要注意的是，如果传入的 iterable 为空数组，Promise.all()将立即解决并返回一个空数组，因为没有需要等待的 Promise。如果 iterable 中包含非 Promise 对象，这些对象将被视为已经解决的 Promise，并以它们自身作为解决值返回。
