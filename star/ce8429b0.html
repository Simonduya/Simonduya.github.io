区别 target 和 currentTarget

在 DOM 事件中，target 和 currentTarget 是两个常用的属性，用于访问事件的目标元素和当前正在处理事件的元素。它们之间的区别如下：

target 属性：target 指向触发事件的目标元素，即实际触发事件的元素。对于事件冒泡阶段，target 始终保持不变。例如，如果在一个按钮上点击，那么 target 将是该按钮元素本身。

currentTarget 属性：currentTarget 指向当前正在处理事件的元素，即事件处理程序所绑定的元素。在事件传播过程中，currentTarget 可能会发生变化。当事件处理程序被调用时，currentTarget 始终是事件监听器所附加到的元素。例如，如果在一个按钮上点击，而该按钮内部嵌套了一个元素，并且事件监听器被附加到按钮上，则 currentTarget 将是按钮元素。

DOM 分级

DOM0 级：指的是最早的 DOM 规范，也称为初始 DOM。它并没有明确定义事件模型，而是在 HTML 标签的属性中直接使用 JavaScript 代码来指定事件处理程序。这种模型简单直观，但功能有限。

DOM1 级：DOM1 级规范于 1998 年发布，引入了基本的文档结构和节点访问方法。它定义了核心的 DOM 结构，如 Document、Element、Node 等，以及操作这些结构的方法和属性。DOM1 级并未引入明确的事件模型，而是在 HTML 标签的属性中使用 DOM0 级风格的事件处理程序。

DOM2 级：DOM2 级规范于 2000 年发布，对 DOM1 级进行了扩展。其中最重要的改进是引入了事件模型，也就是我们之前提到的 addEventListener()和 removeEventListener()方法。DOM2 级事件模型更灵活，允许同时关联多个事件处理程序，可以在捕获和冒泡阶段处理事件，提供了更多的事件类型和功能。

DOM3 级：DOM3 级规范于 2004 年发布，进一步扩展了 DOM 的功能。它引入了一些新的模块，包括 DOM 加载和保存、DOM 验证、XPath、事件模型的扩展等。DOM3 级事件模块引入了更多的事件类型和方法，如 focus、blur、input 等。

DOM0 级标准事件是指在早期的 JavaScript 和 DOM 规范中定义的事件模型。DOM（Document Object Model）是一种用于表示和操作 HTML 和 XML 文档的标准编程接口。

在 DOM0 级标准事件模型中，事件是由浏览器中的元素触发的交互性动作。它们允许开发人员对用户的操作做出响应，如单击、双击、鼠标移动等。

DOM0 级标准事件使用了一种简单的事件处理方式，开发人员可以直接在 HTML 标签的属性中指定事件处理程序。例如，在一个按钮元素上定义一个点击事件处理程序可以使用以下代码：

DOM0 级标准事件模型比较简单，事件处理程序直接与 HTML 元素相关联。然而，这种方式存在一些限制，比如只能同时关联一个事件处理程序，不够灵活等。后续的 DOM 规范版本（如 DOM2 级和 DOM3 级）引入了更强大和灵活的事件模型，提供了更多的事件类型和事件处理方式

DOM2 级事件：

addEventListener()：通过此方法可以向元素添加事件监听器，可以同时关联多个事件处理程序。
removeEventListener()：用于从元素中移除事件监听器。
click：当用户点击元素时触发的事件。
mouseover：当鼠标指针移动到元素上方时触发的事件。
keydown：当用户按下键盘上的某个键时触发的事件。

DOM3 级事件：

focus：当元素获得焦点时触发的事件。
blur：当元素失去焦点时触发的事件。
input：当元素的值发生变化时触发的事件，适用于文本框、文本区域等输入元素。
transitionend：当 CSS 过渡效果完成后触发的事件。
wheel：当鼠标滚轮滚动时触发的事件。

这些事件只是示例，DOM2 级和 DOM3 级规范中定义了更多的事件类型和方法，以及更复杂的事件处理机制，使开发人员能够更灵活地处理各种用户交互行为。

总结

target 表示当前触发事件的元素

currentTarget 是绑定处理函数的元素

DOM 事件有分级

  1. 使用 indexOf() 方法：遍历数组，在新数组中只添加原数组中不存在的元素。
  1. 使用 includes() 方法：遍历数组，在新数组中只添加原数组中不存在的元素。

  1. 使用 reduce() 方法：使用 reduce() 方法遍历数组，在新数组中只添加原数组中不存在的元素。

实现 instanceof 的原理

instanceof 的原理是基于对象的原型链。当你使用 instanceof 操作符来判断对象是否是某个构造函数的实例时，它会沿着对象的原型链向上查找，直到找到与构造函数的原型对象相等的原型，或者到达原型链的尽头（null）。

基本的 instanceof 实现可以通过迭代对象的原型链来进行判断。它会依次获取对象的原型，与给定构造函数的原型进行比较。如果找到匹配的原型，返回 true，否则返回 false。

  1. proto 参数：这是一个必需的参数，表示新对象的原型对象。它可以是任何对象或 null。新对象将继承指定原型对象的属性和方法。

  2. propertiesObject 参数（可选）：这是一个可选的参数，用于定义新对象的属性。它是一个对象，其中每个属性都是一个描述符对象。每个描述符对象可以具有以下属性：

    • value：属性的值，默认为 undefined。
    • writable：属性是否可写，默认为 false。
    • enumerable：属性是否可枚举，默认为 false。
    • configurable：属性是否可配置，默认为 false。

    你可以使用 propertiesObject 参数来定义新对象的初始属性。
