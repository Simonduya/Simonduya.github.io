真实 DOM 渲染

在过去，我们编写的 HTML 文档会被浏览器解析成 DOM 树，最终呈现为页面上的各个元素。

虚拟 DOM 的优势

虚拟 DOM（Virtual DOM）是一种对真实 DOM 的抽象概念，目前被广泛引入到三大前端框架中。

1. 简化 DOM 操作

直接操作真实 DOM 元素的操作相对繁琐。例如，如果要复制一个<div>元素，需要复制该元素的各种属性，这会显著增加代码复杂性。

虚拟 DOM 通过 JavaScript 对象的形式表示 DOM 节点（称为 VNode），这样可以更轻松地进行操作和处理，避免了直接操作庞大的真实 DOM 树。

2. 高效的差异计算

在进行 DOM 更新时，常常需要比较两个不同状态下的 DOM 结构，以确定需要更新哪些部分。直接对比两个真实 DOM 树的复杂度较高。

虚拟 DOM 使得比较两个 VNode 树变得更加高效，因为它们是纯 JavaScript 对象，可以进行快速的比较，减少了性能开销。这一过程通常被称为”diff 算法”。

3. 降低浏览器回流

对真实 DOM 的频繁操作可能导致浏览器的回流（reflow），这是一种性能开销较大的操作，可能导致页面重新排版。虚拟 DOM 的存在可以减少对真实 DOM 的直接操作，从而降低回流的发生。

4. 跨平台兼容性

虚拟 DOM 的抽象层使得前端应用更容易跨不同平台和环境使用。因为虚拟 DOM 是基于 JavaScript 的，它可以在浏览器、移动应用和服务器端渲染等多种环境中使用，使得开发更加灵活和可维护。

综上所述，虚拟 DOM 的引入为前端开发带来了诸多优势，简化了操作、提高了性能，同时也提高了跨平台的可移植性。这是为什么现代前端框架广泛采用虚拟 DOM 的主要原因之一。

虚拟 DOM 的渲染过程

虚拟 DOM 的渲染过程通常可以描述如下：

  1. 编写模板（Template）：前端开发人员编写模板，这个模板通常包含 HTML 结构和模板语法，例如 Vue 中的模板或 React 中的 JSX。

  2. 模板编译（Template Compilation）：模板会被编译器（Compiler）处理，将模板转换为可执行的渲染函数。这个过程通常发生在构建阶段，例如在 Vue 中使用的单文件组件编译器或者 React 中的 Babel。

  3. 调用渲染函数（Invoke Render Function）：在运行时，编译后的渲染函数会被调用，这个函数生成虚拟节点（VNode）树。

  4. 虚拟节点构建（VNode Creation）：渲染函数内部会调用 h 函数（或 createElement 函数），用于创建虚拟节点（VNode）。这些虚拟节点代表了页面的结构和内容。

  5. 虚拟 DOM 构建（Virtual DOM Creation）：一系列的虚拟节点会被组合成虚拟 DOM 树，这个树形结构代表了应用程序的当前状态。

  6. 虚拟 DOM 与真实 DOM 的比对（Virtual DOM Diffing）：当应用程序的状态发生改变时，新的虚拟 DOM 树会与之前的虚拟 DOM 树进行比对，找出差异（Diffing）。

  7. 差异更新（Patch Updates）：找到差异后，渲染器（Renderer）会将这些差异应用到真实的 DOM 上，这个过程通常被称为“补丁更新”，它只会更新真实 DOM 中实际发生变化的部分，而不是整个页面。

  8. 浏览器渲染（Browser Rendering）：最终，更新后的真实 DOM 会被浏览器渲染到屏幕上，用户可以看到更新后的界面。

虚拟 DOM 的优势在于它将真实 DOM 的操作和更新抽象成了一个高效的过程，通过比对差异，只更新必要的部分，从而提高了性能和用户体验。这种机制在流行的前端框架（如 Vue 和 React）中得到广泛应用，以实现响应式和高效的界面更新。

template 被 compiler 编译为 render 函数, 然后调用 render 函数, render 函数中又会再次调用 h 函数, h 函数返回虚拟节点 VNode, 一个个 VNode 形成虚拟 DOM,再通过渲染器将虚拟 DOM 转为真实的 DOM, 最后交给浏览器渲染

三大核心系统

三大核心系统模块是现代前端框架中的重要组成部分，它们协同工作以实现整个应用程序的渲染和响应式更新：

  1. 编译器模块（Compiler Module）：编译器负责将开发人员编写的模板（Template）转换为可执行的渲染函数。这个渲染函数通常会调用 h 函数（或 createElement 函数），生成虚拟节点（VNode）树，并最终返回树的根节点。这个模块在构建阶段运行，例如，在 Vue.js 中，Vue Loader 会处理单文件组件并通过编译器将其转换为 JavaScript 代码。

  2. 运行时模块（Runtime Module，或称为 Renderer 模块）：运行时模块负责将虚拟节点树渲染为真实的 DOM 元素，并将其呈现在浏览器中。在这个过程中，运行时模块会使用响应式系统来监听数据变化，一旦数据发生变化，它将触发重新渲染，并将新的虚拟节点与旧的虚拟节点进行比对，然后更新真实的 DOM。这个模块使得应用程序能够动态响应数据的变化，实现了双向数据绑定等功能。

  3. 响应式模块（Reactivity Module）：响应式模块是前端框架的核心之一，它负责建立数据的监听机制。当数据发生变化时，响应式系统会自动通知相关的依赖，并执行相应的代码来处理这些变化。在响应式系统内部，通常会有一个 patch 函数，用于执行差异比对（Diffing）算法，以确定哪些虚拟节点需要更新，然后将这些变化应用到真实的 DOM 上。这个模块使得开发人员可以方便地管理应用程序的状态和数据，以及实现数据驱动的视图更新。

综合而言，这三大核心系统模块共同构成了现代前端框架的基础架构，使得开发人员能够更容易地构建高性能、响应式的用户界面。这种模块化和分层的设计有助于提高代码的可维护性和可扩展性，同时实现了前端应用程序的复杂功能和动态性。
